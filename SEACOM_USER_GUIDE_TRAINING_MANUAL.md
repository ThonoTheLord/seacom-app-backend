# Seacom User Guide and Training Manual

## Version
- Version: 1.1
- Last updated: 2026-02-25

## Purpose
This is the facilitator manual for delivering onboarding sessions.  
For user-facing role instructions, use `SEACOM_USER_TRAINING_GUIDE.md`.

## Session Plan (Recommended)

### Module 1: Platform Orientation (20 min)
- Explain mission-critical workflows: incidents, tasks, SLA, escalation.
- Map user roles to expected decisions and actions.

### Module 2: Navigation and Basic Operations (30 min)
- Login and profile setup.
- Dashboard navigation.
- Notifications and status updates.

### Module 3: Role Simulation (45 min)
- NOC scenario: detect risk, assign, escalate.
- Technician scenario: acknowledge, update progress, complete with evidence.
- Manager scenario: resolve escalation and rebalance workload.

### Module 4: Incident Drill (30 min)
- Simulate a high-priority incident with timeline pressure.
- Evaluate communication quality, ownership clarity, and SLA awareness.

### Module 5: Assessment and Sign-off (15 min)
- Practical checklist validation per role.
- Capture gaps and assign follow-up coaching.

## Trainer Checklist
- Use real system screens where possible.
- Enforce status accuracy and update discipline.
- Require evidence attachment before closure in simulations.
- Test escalation quality: who, why, impact, and requested action.

## Completion Criteria
- User can complete role-specific critical path without supervision.
- User can identify when to escalate.
- User maintains high-quality operational notes.
- User demonstrates correct SLA awareness behavior.

## Post-Training Follow-up
- Week 1: daily spot checks.
- Week 2: targeted coaching on recurring errors.
- End of month: re-certification drill for critical workflows.
